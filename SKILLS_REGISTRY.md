## Usage Patterns

**Quick task** → Mention skill by name in main session
**Complex feature** → Spawn sub-agent with specific skills
**Research/spike** → Use web_search first, then skill if needed
**Parallel work** → Multiple sub-agents, different skill sets

## Anti-Patterns
❌ Loading all skills "just in case"
❌ Never using sub-agents for domain work
❌ Not updating this registry when skills change
